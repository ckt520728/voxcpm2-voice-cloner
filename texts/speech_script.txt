建構 Notion 第二大腦：從踩坑到完整模板系統

今天和 Hermes Agent 一起建構 Notion 第二大腦系統的完整記錄，包含踩過的坑、教訓、以及最終建立的模板系統。

背景是這樣的——我是一位臨床腎臟科醫師，同時在進行認知神經科學的博士研究，主題是用機器學習分析腦波，來做失智症的早期診斷。知識管理的需求非常複雜，包括學術論文、專業書籍、部落格文章、以及 YouTube 影片。過去的學習內容散落在 Notion 不同頁面中，沒有統一格式。今天的目標，就是建立一套標準化的筆記模板。

今天踩到了一個非常重要的坑：Notion API 刪除 blocks 會導致資料永久遺失。當我請 Hermes Agent 重新整理演講筆記頁面時，它的做法是先刪除頁面上所有現有的四十個 blocks，再重新建立新的結構性 blocks。結果所有 Toggle 裡面的內容，包括 AI 對話和研究筆記，全部消失了，link to page 的關聯也被切斷。

這個教訓非常深刻：絕對不要在保存內容之前刪除現有的 blocks。正確的做法永遠是在現有 blocks 之間插入新的內容，而不是刪除後重建。

最終，我建立了四種內容類型的模板，並且共用一套統一的標籤體系。標籤分為三個維度：研究領域、重要性等級、以及閱讀狀態。

四種模板分別是：書籍模板，包含基本資訊、一句話摘要、Key Takeaways、金句、章節筆記和 AI 對話；學術論文模板，包含 BibTeX、研究問題、方法、關鍵發現、批判性問題和程式碼；部落格文章模板，包含核心觀點、重點筆記、我的看法和可行動項目；以及 YouTube 影片模板，包含核心內容、重點筆記，以及附有時間戳的問題與想法。

安全重構的正確姿勢只有幾個原則：在現有 blocks 之間插入新的 blocks；只修改 block 的 properties；操作前完整讀取所有內容並備份。絕對不要做的就是刪除全部再重建。

這次也建立了一個叫做 notion-second-brain-builder 的技能包，包含完整工作流、安全規則、四種模板規格，以及 Notion API 的常見陷阱，希望未來不再踩同樣的坑。
